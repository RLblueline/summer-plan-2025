Operating brief

Insanely specific summer plan.

This version tells you exactly what to do each weekday, what math topic to study, what Python topic to practice, what project task to execute, what repo to update, and what deliverable should exist at the end of each week.

  • Obstacle-avoiding or line-following robot using Arduino-class hardware, motor driver, sensors, and a basic chassis.
  • One classical ML project first, then one OpenCV/computer-vision integration project as the flagship.
  • 4.5 morning hours max so afternoons stay mostly free.

Base workday

Exact daily template

Every weekday follows this structure unless a note says otherwise.

08:00 – 08:30

Start-up

Wake up, eat breakfast, stretch or walk 5–10 minutes, and open only the tools needed for the day.

08:30 – 09:15

Math topic

One exact topic per day, 5–10 practice problems, and one short note on what still feels weak.

09:15 – 10:00

Python coding topic

One exact coding concept per day plus one mini script, notebook cell group, or exercise.

10:00 – 10:15

Break

Stand up, water, no scrolling, and come back with the next project subtask chosen already.

10:15 – 12:30

Main build block

Hardware, AI, CV, debugging, filming, documentation, and repo updates. Final 20–30 minutes always go to notes and commits.

Parts, software, repos

Exact project stack

This is the default stack the rest of the schedule assumes so you do not have to keep deciding.

Project 1: Robotics build

Goal: one autonomous robot with reliable sensor-based behavior.

  • Arduino, motor driver, 2 DC motors, wheels, chassis, caster wheel, battery pack, breadboard, jumper wires, ultrasonic sensor or line sensors, switch, zip ties/hot glue.
  • Arduino IDE, serial monitor, optional simple CAD sketch or paper sketch.
  • summer-robotics-build with folders: code, media, notes, wiring.
  • Final demo video, wiring photos, code, README, and one lessons-learned section.

Project 2: AI/ML build

Goal: one practical Python ML project that goes from raw data to evaluated result.

  • Python, Jupyter or VS Code notebooks, pandas, numpy, matplotlib, scikit-learn.
  • summer-ai-build with folders: data-notes, notebooks, plots, reports.
  • Notebook or script, comparison of two models, plots, and plain-English explanation.
  • Tabular classification/regression or a simple CV/object-detection prototype.

Project 3: Flagship integration

Goal: one project that clearly combines AI/CV thinking with hardware or a visibly real system.

  • Pi or laptop camera/OpenCV setup, existing robot or actuator system, plus Python/OpenCV.
  • summer-flagship-build (or a flagship folder inside summer-ai-build if you want fewer repos).
  • Stable demo clip, architecture diagram, README, and a short "future improvements" list.
  • Object-tracking robot, vision-assisted follower, or camera-triggered embedded behavior.
